# E-commerce Core: Каталог товаров

**Проект представляет собой ядро для интернет-магазина**, реализующее базовые сущности: категории и товары. В дальнейшем на основе этой логики можно построить любой интерфейс — от веб-сайта до Telegram-бота.

---

## Реализованные функции

### Классы

#### `Product`
Класс товара. Содержит:
- `name`: название товара  
- `description`: описание  
- `price`: цена   
- `quantity`: количество на складе 

#### `Category`
Класс категории товаров. Содержит:
- `name`: название категории  
- `description`: описание категории  
- `products`: список объектов `Product`  
- `category_count`: общее количество созданных категорий 
- `product_count`: общее количество продуктов во всех категориях

Также созданы два класса наследников класса 
Product с расширенными атрибутами: 
- «Смартфон» (Smartphone)
- «Трава газонная» (LawnGrass)

Реализованы методы:
- сложения (чтобы можно было складывать товары только из одинаковых классов продуктов).
- добавления продукта (чтобы не было возможности добавить вместо продукта или его наследников любой другой объект).


---

### Загрузка данных из JSON

Функция `load_data_from_json(filepath)`:
- Загружает структуру категорий и товаров из JSON-файла
- Создаёт объекты `Category` и `Product`
- Обрабатывает ошибки (например, отсутствие поля `price`)

Файл с данными хранится в папке [`/data`](./data).

---

### Тестирование

Тесты написаны с использованием `pytest` и покрывают:
- Инициализацию объектов `Product` и `Category`
- Подсчёт категорий и продуктов
- Обработку нестандартных и пустых данных
- Загрузку из JSON-файла
- Ограничения сложения добавления продуктов

## Установите зависимости:
`poetry install`
## Запустите тесты:
`pytest`
## Получите отчёт о покрытии:
`pytest --cov=src --cov-report=html`

HTML-отчёт будет создан в папке htmlcov/. Откройте файл htmlcov/index.html в браузере.
Процент покрытия составляет 96%
